Appendix A

Glossary

A

Archetype
Universal patterns or images in the collective unconscious, as described by Carl Jung. In this book, archetypes represent different aspects of the programmer's journey and Rust's features.
Async/Await
Syntax for writing asynchronous code that looks like synchronous code. async marks a function as asynchronous; .await pauses execution until a Future completes.
Associated Type
A type placeholder in a trait that must be specified by implementations. Allows traits to work with specific types without using generics.

B

Borrowing
Creating a reference to data without taking ownership. Rust's borrow checker ensures references are valid and follow the rules: one mutable OR any number of immutable references.
Box
A smart pointer that allocates data on the heap. Box<T> owns the data and automatically frees it when dropped.
Borrow Checker
Rust's compile-time system that enforces ownership and borrowing rules, preventing data races and memory errors.

C

Closure
An anonymous function that can capture variables from its enclosing scope. Closures implement the Fn, FnMut, or FnOnce traits.
Crate
A compilation unit in Rust‚Äîeither a binary (executable) or library. The crate root is main.rs or lib.rs.
Cargo
Rust's build system and package manager. Handles compiling code, downloading dependencies, and building projects.

D

Drop Trait
A trait for running cleanup code when a value goes out of scope. Implements the RAII (Resource Acquisition Is Initialization) pattern.
Dangling Pointer
A pointer that references memory that has been freed or is otherwise invalid. Rust's borrow checker prevents these at compile time.

F

Fat Pointer
A pointer that carries additional metadata, such as a vtable pointer (for trait objects) or length (for slices). Twice the size of a regular pointer on 64-bit systems.
Future
A value that may not be ready yet. The core abstraction of async Rust‚Äîrepresents an asynchronous computation that produces a value.

G

Generic
A type or function parameterized over other types. Enables code reuse while maintaining type safety. Rust uses monomorphization to generate concrete implementations.
Generic Associated Type (GAT)
Associated types that can have their own generic parameters, enabling more flexible trait design.

H

Heap
Memory for dynamically allocated data with arbitrary lifetime. Managed manually or through smart pointers like Box, Rc, or Arc.

I

Individuation
Jung's concept of becoming one's true self by integrating all aspects of the psyche‚Äîincluding the unconscious. The journey this book guides you through.
Iterator
A trait for types that can produce a sequence of values. Enables lazy evaluation and composable data processing through methods like map, filter, and fold.
Interior Mutability
A design pattern allowing mutation through immutable references when runtime checks ensure safety. Implemented by RefCell, Mutex, and atomic types.

L

Lifetime
The scope during which a reference is valid. Rust tracks lifetimes to prevent dangling pointers. Denoted by tick syntax: 'a.

M

Monomorphization
The process of generating specific implementations of generic code at compile time. Enables zero-cost abstractions‚Äîgenerics have no runtime overhead.
Module
A namespace that organizes code within a crate. Declared with mod, visibility controlled with pub.
Move Semantics
Transferring ownership of data from one variable to another. The source can no longer be used after the move.

O

Ownership
Rust's core memory management principle: each value has a single owner. When the owner goes out of scope, the value is dropped.
Option
An enum representing a value that may or may not exist: Some(T) or None. Rust's alternative to null pointers.

R

RAII
Resource Acquisition Is Initialization‚Äîtying resource lifetime to object lifetime. Resources are acquired in constructors and released in destructors (Drop).
Result
An enum for operations that may fail: Ok(T) for success or Err(E) for failure. Forces explicit error handling.
Reference
A non-owning pointer to data. Immutable (&T) or mutable (&mut T). Always valid and never null.

S

Shadow (Archetype)
The unconscious aspect of the personality, containing repressed weaknesses and instincts. In Rust, represented by unsafe code.
Send Trait
A marker trait indicating a type can be safely transferred between threads. Most types are Send.
Sync Trait
A marker trait indicating a type can be safely shared between threads (via references). &T is Send if T is Sync.
Stack
Memory for function-local data with automatic allocation and deallocation. Fast but limited in size and scope.
Smart Pointer
A data structure that acts like a pointer but has additional metadata and capabilities. Examples: Box, Rc, Arc, RefCell.
Self (Archetype)
The center of the psyche, representing the unified personality. The goal of individuation‚Äîthe integration of all archetypes.

T

Trait
A collection of methods defined for an unknown type. Rust's mechanism for abstraction and polymorphism‚Äîsimilar to interfaces in other languages.
Trait Object
A fat pointer to a trait instance, enabling dynamic dispatch. Written as dyn Trait. Enables runtime polymorphism.
Tuple
A fixed-size ordered list of values of potentially different types. Written as (T, U, V).
Type Inference
The compiler's ability to deduce types from context, reducing the need for explicit type annotations.

U

Unsafe
Code blocks or functions marked with unsafe where the programmer takes responsibility for safety. Enables raw pointer dereferencing, FFI, and other low-level operations.
Undefined Behavior (UB)
Operations that violate Rust's safety guarantees. The compiler assumes UB never happens; if it does, anything can happen (crashes, security vulnerabilities).

V

Vtable
A table of function pointers used for dynamic dispatch with trait objects. Enables calling the correct method implementation at runtime.

W

Waker
A mechanism in async Rust for notifying the executor that a Future should be polled again. Part of the async runtime infrastructure.

Z

Zero-Cost Abstraction
An abstraction that has no runtime overhead compared to hand-written lower-level code. Rust's iterator chains, generics, and trait methods compile to efficient machine code.
