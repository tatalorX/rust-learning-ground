Five Powers of Unsafe

Unsafe Rust grants five capabilities:

  • Dereference raw pointers
  • Call unsafe functions
  • Access or modify mutable static variables
  • Implement unsafe traits
  • Access fields of unions

Raw Pointer Dereferencing

let mut num = 5;
let r1 = &num as *const i32;
let r2 = &mut num as *mut i32;

unsafe {
    println!("r1 is: {}", *r1);
    *r2 = 10;
    println!("r2 is now: {}", *r2);
}

Implementing Safe Abstractions

The pattern: unsafe implementation, safe interface:

pub struct SafeBuffer<T> {
    ptr: *mut T,
    len: usize,
    cap: usize,
}

impl<T> SafeBuffer<T> {
    // Safe public interface
    pub fn new() -> Self {
        // ... safe initialization
    }
    
    pub fn get(&self, index: usize) -> Option<&T> {
        if index < self.len {
            unsafe { Some(&*self.ptr.add(index)) }
        } else {
            None
        }
    }
}

Unsafe Traits

Some traits are unsafe to implement because the compiler cannot verify their guarantees:

unsafe trait Zeroable {
    fn zero() -> Self;
}

unsafe impl Zeroable for u32 {
    fn zero() -> Self { 0 }
}

unsafe impl<T: Zeroable> Zeroable for [T; 4] {
    fn zero() -> Self {
        [T::zero(), T::zero(), T::zero(), T::zero()]
    }
}
