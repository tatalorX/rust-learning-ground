Visibility

By default, everything is private. Explicitly mark what should be public:

mod front_of_house {
    pub mod hosting {           // Module is public
        pub fn add_to_waitlist() {}  // Function is public
        fn seat_at_table() {}        // Function is private
    }
}

Workspaces

For multi-crate projects, use workspaces:

# Cargo.toml (workspace root)
[workspace]
members = [
    "my-app",
    "my-lib",
    "common-utils",
]

Each member is an independent crate with its own Cargo.toml, but they share a Cargo.lock and build artifacts.

ğŸ”¬ Surgical Precision: Module Privacy Rules

Rust's visibility system is module-based, not type-based:

// In module foo:
pub struct Bar { pub x: i32, y: i32 }  // y is private

// In another module:
use foo::Bar;
let b = Bar { x: 1, y: 2 };  // ERROR: y is private
let b = Bar { x: 1 };        // OK (if Bar has default for y)

// Privacy is checked at the module boundary,
// not based on where the code is written.
