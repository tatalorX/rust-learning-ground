The surgeon does not cut without diagnosis. The performance engineer does not optimize without measurement. This interlude provides the diagnostic tools‚Äîthe stethoscope and scalpel of performance work.

The Measurement Tools

Criterion.rs ‚Äî Statistical Benchmarking

use criterion::{black_box, criterion_group, criterion_main, Criterion};

fn fibonacci(n: u64) -> u64 {
    match n {
        0 => 1,
        1 => 1,
        n => fibonacci(n - 1) + fibonacci(n - 2),
    }
}

fn criterion_benchmark(c: &mut Criterion) {
    c.bench_function("fib 20", |b| {
        b.iter(|| fibonacci(black_box(20)))
    });
}

criterion_group!(benches, criterion_benchmark);
criterion_main!(benches);

perf ‚Äî Linux Profiling

# Compile with debug symbols and frame pointers
$ RUSTFLAGS="-C force-frame-pointers=yes" cargo build --release

# Record performance data
$ perf record --call-graph=dwarf ./target/release/myapp

# Generate flamegraph
$ perf script | inferno-collapse-perf | inferno-flamegraph > flame.svg
üî¨ Surgical Precision: Reading Flamegraphs

A flamegraph shows where your program spends time. The x-axis is not time‚Äîit is population. The wider a function, the more time it consumes. The y-axis is stack depth. Look for plateaus‚Äîthese are functions consuming significant time. Look for spikes‚Äîthese are deep call chains that may indicate inefficiency.

Optimization Techniques

1. Avoid Allocations

// Slow: allocates a Vec
let sum: i32 = (0..1000).collect::<Vec<_>>().iter().sum();

// Fast: no allocation
let sum: i32 = (0..1000).sum();

2. Use Release Mode

# Debug mode: fast compile, slow runtime
$ cargo run

# Release mode: slow compile, fast runtime
$ cargo run --release

3. Profile-Guided Optimization (PGO)

# Build with instrumentation
$ RUSTFLAGS="-C profile-generate=/tmp/pgo" cargo build --release

# Run representative workload
$ ./target/release/myapp

# Build optimized binary
$ RUSTFLAGS="-C profile-use=/tmp/pgo" cargo build --release
